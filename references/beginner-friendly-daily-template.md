# 新手友好版每日内容模板

本文件用于生成适合 Python 零基础或基础薄弱学习者的每日学习内容。目标是让学习者能照着做、能跑通、能理解为什么这么做，而不是把标准版内容简单扩写。

## 适用场景

- 用户要求“生成 Day N 新手友好版”。
- 用户要求“零基础版 / 保姆级讲解 / 新手友好感”。
- 用户要求把课程内容保存为 `标准版` 和 `新手友好版` 两份文件。
- 用户反馈概念没懂、代码没跑通、需要从零讲起。

## 输出结构

新手友好版每日内容按以下结构输出。下面是结构说明，不是要原样输出的代码块：

- `# 📅 Day N：主题（新手友好版）`
- 元信息：周期、主题、目标、适合人群。
- `## 🧭 先别急着写代码！搞懂这三个比喻`
- 三个比喻小节：每个抽象概念都对应一个生活场景。
- `## 🎯 今天要完成的事（只做三件事！）`
- 三个以内的核心任务。
- `## ⏰ 分步学习路线（总共 X 小时）`
- 每一步写清预计时间、操作、成功检查点和失败排查。
- `## 💻 今天的核心代码`
- 代码块前写 `关联知识：[[知识点 A]]、[[知识点 B]]、[[知识点 C]]`。
- 代码块后写运行方式、观察点和报错排查。
- `## 🧩 关键概念对照表`
- 表格列：概念、新手解释、生活比喻、Agent 场景。
- `## ❓ 新手常见问题`
- 至少 3 个新手高频问题。
- `## ✅ 今日验收清单`
- 使用 checkbox 写出可验证结果。
- `## 📥 明日同步接口`
- 使用固定 Day N 学习反馈模板结尾。

## 内容颗粒度

- 每天只讲 2-3 个核心任务。
- 每个抽象概念必须配生活比喻。
- 每一步都要说明“做什么、为什么做、看到什么算成功、失败时怎么排查”。
- 代码尽量短，但必须完整可运行。
- 不把高级概念塞进新手友好版；如果必须提到，只用一句话解释，不展开。

## 双链策略

新手友好版使用克制双链：

- 标题或首段中给核心概念加双链。
- `关联知识：` 中列出 3-5 个稳定双链。
- 验收题或验收清单中可以少量加双链。
- 代码块内部禁止双链。
- 文件名、命令、环境变量、API 路径用反引号，不加双链。

推荐优先使用这些稳定标题：

- `[[Python 虚拟环境]]`
- `[[环境变量与 API Key]]`
- `[[.gitignore 配置]]`
- `[[JSON 文件读写]]`
- `[[Python 函数]]`
- `[[异常处理]]`
- `[[HTTP 请求结构]]`
- `[[HTTP 状态码]]`
- `[[OpenAI-compatible API]]`
- `[[LLM messages 结构]]`
- `[[LLMClient 封装]]`
- `[[对话历史持久化]]`

## 文件成对输出规则

当用户要求生成课程文件时，同一个 Day 输出两个文件路径：

```text
标准版/DayN-主题.md
新手友好版/DayN-主题-新手友好版.md
```

标准版内容保持简洁、任务导向；新手友好版内容更长、更细、更像手把手教程。

示例：

```text
标准版/Day1-Python-工程环境与函数基础.md
新手友好版/Day1-Python工程环境与函数基础-新手友好版.md
```

## Day 1-Day 3 风格样例

- Day 1：用“私人厨房”解释 `[[Python 虚拟环境]]`，用“保险箱”解释 `[[环境变量与 API Key]]`，用“拒收清单”解释 `[[.gitignore 配置]]`。
- Day 2：用“快递系统”解释 `[[HTTP 请求结构]]`，用“点餐窗口”解释 API，用“签收状态”解释 `[[HTTP 状态码]]`。
- Day 3：用“电话客服中心”解释 `[[OpenAI-compatible API]]`，用“微信群里的三个人”解释 `[[LLM messages 结构]]`，用“随机程度旋钮”解释 temperature。

## 质量检查

生成后自查：

- 是否有三个生活比喻。
- 是否只聚焦三件事以内。
- 是否每一步都有成功检查点。
- 是否有常见问题和验收清单。
- 代码块内部是否没有 `[[...]]`。
- 双链是否克制且使用稳定命名。
